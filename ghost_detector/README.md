# 👻 幽灵探测器 Ghost Detector

> 一个用真实传感器数据驱动"伪科学叙事"的交互装置 —— 红外、温度、光敏的读数，被转译成灵异事件的证据。

---

## 核心思路

```
真实物理变化          →     奇幻叙事解读

红外突然感应到人体     →     "侦测到高速移动实体"
温度瞬间下降 0.5°C     →     "三级冷点正在形成"
暗处光敏捕捉到微光波动  →     "2.4GHz EMF 异常脉冲"
三个传感器同时异常     →     "⚠ A类完整显灵事件！"
```

**不是随机乱报，而是传感器真的检测到变化才触发**，让人觉得"它确实捕捉到了什么"。

## 核心玩法

### 事件分级系统

| 等级 | 触发条件 | LCD 反应 |
|:--|:--|:--|
| 🟢 **平静** | 三个传感器都稳定 | 暗屏 + 雷达慢速扫描 + "Standby..." |
| 🟡 **疑似** | 单个传感器小波动 | 屏幕边缘泛黄 + 文字提示 |
| 🟠 **警告** | 两个传感器同时异常 | 屏幕闪烁 + 红色感叹号 + 具体描述 |
| 🔴 **显灵** | 红外+温度+光敏同时异常 | 全屏红光 + 大字警告 + 蜂鸣器 |

### 让人信服的关键设计

1. **联动触发** —— 必须是真实物理变化才触发（挥手 → 红外+温度差 = 联动），不会对着空房间乱叫
2. **安静时也安静** —— 环境稳定时设备处于休眠扫描态，不刷存在感
3. **科学包装** —— UI 用 μT、mG、GHz 这些单位，界面像专业测量设备
4. **交叉验证** —— 两个传感器同时 = 更高级别，增加可信度

## 程序结构

```
ghost_detector/
  README.md           # 本文档
  config.py           # 引脚、阈值、WiFi/MQTT 配置
  sensors.py          # 传感器初始化 + 读数封装
  narrative.py        # 事件文案库（20-30 条）
  display.py          # LCD 界面渲染（雷达 / 数据 / 日志）
  ghost_detector.py   # 核心：变化率检测 + 事件分级 + 叙事匹配
  main.py             # 主循环
```

### 核心逻辑伪代码

```python
# 每秒采样一次
temp_delta = 当前温度 - 上次温度
light_delta = 当前光照 - 上次光照  # 取绝对值
pir_active = 红外传感器 == 1

# 判断几个传感器同时异常
异常数 = 0
if 温度变化 > 0.3°C:  异常数 += 1
if 光照波动 > 阈值:    异常数 += 1
if 红外触发:          异常数 += 1

# 分级 + 选文案
if   异常数 == 0 → 平静 → LCD 显示雷达扫描
elif 异常数 == 1 → 疑似 → LCD 提示 + 边缘黄光
elif 异常数 == 2 → 警告 → LCD 闪烁 + 事件描述
elif 异常数 >= 3 → 显灵 → 全屏红光 + 蜂鸣器
```

## 扩展玩法（后续迭代）

- 🔮 **问答模式**：按按钮问一个问题（心里默想），设备扫描 5 秒后显示 YES / NO / UNCERTAIN
- 🌙 **夜间守护模式**：睡前开启，夜里红外被触发就记录时间——"凌晨 3:17 检测到活动"，第二天看日志
- 🔊 **Geiger 音效**：每当事件触发，蜂鸣器发出咔嗒声，越高级别频率越高
- 📡 **MQTT 远程告警**：显灵事件推送通知到手机——"你房间有异常活动，速查看"

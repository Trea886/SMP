# ⚡ 反应挑战机 Reaction Challenge

> ESP32 桌面反应力测试装置 + 赛博朋克 Web 全球排行榜
> PAJ7620 手势答题 · COMBO 连击 · MQTT 实时排名

---

## 一、项目概述

桌面上一台 ESP32 小装置，LCD 屏幕随机出题，你用手势作答。题目分两种类型混合出题：

- **方向题（60%）**：屏幕显示方向箭头（^ v < >），挥出对应手势
- **距离题（40%）**：屏幕显示 FORWARD 或 BACK，前推或后退手势

答对 COMBO 越叠越高，答错或超时游戏结束。分数自动上传 MQTT，Web 排行榜实时刷新。

上电后三页自动流转：**标题页（5s）→ 游戏中 → 结算停止**。

| | |
|:--|:--|
| 🎮 **两种题型** | 方向题 60% + 距离题 40%，随机混合 |
| 🤖 **传感器** | PAJ7620 手势识别（使用 6 种手势） |
| ⚡ **COMBO** | ×1 → ×2 → ×3 → ×5 → ×10，最高 ×10 封顶 |
| 🏆 **反方向陷阱** | 第 12 题起 30% 概率出现，须挥反方向 |
| 🌐 **联网 PK** | MQTT 上传分数 → Web 排行榜 |
| 📦 **无线便携** | WiFi 联网，锂电池供电 |

## 三、游戏机制

上电后进入标题页，5 秒后自动开始游戏。每题有倒计时进度条（颜色随剩余时间从绿→黄→红），每题开始有 **1 秒宽限期**防止误触发。

### 答题规则

| 题型 | 屏幕显示 | 正确手势 |
|:--|:--|:--|
| 方向题 · 普通 | ^ v < > 箭头 +「立即挥手」 | 挥对应方向 |
| 方向题 · 陷阱 | ^ v < > 箭头 +「!!反转!!」红色警告 | 挥**反方向** |
| 距离题 · FORWARD | 「前推」+ 手靠近提示 | 前推 |
| 距离题 · BACK | 「后退」+ 手远离提示 | 后退 |

> 陷阱从第 12 题起出现，概率 30%。

### 时限与难度

```
timeout = max(2000, 4000 − (题号 ÷ 6) × 200)  ms
```

| 题号 | 时限 | 难度星级 |
|:--|:--|:--|
| 第 1-5 题 | 4.0 秒 | ★☆☆☆☆ |
| 第 6-11 题 | 3.8 秒 | ★★☆☆☆ |
| 第 12-17 题 | 3.6 秒（陷阱出现） | ★★★☆☆ |
| 第 18-23 题 | 3.4 秒 | ★★★★☆ |
| 第 24 题起 | 3.2 秒起，最低 2.0 秒 | ★★★★★ |

### COMBO 计分

| COMBO | 倍率 | 单题得分 |
|:--|:--|:--|
| 0 – 1 | ×1 | +10 |
| 2 | ×2 | +20 |
| 3 | ×3 | +30 |
| 4 | ×5 | +50 |
| 5+ | ×10 | +100 |

答错或超时 COMBO 清零，游戏结束。

## 四、LCD 界面

三页面流转，中文 16px 点阵渲染（`fonts.py` 字库 + `printChinese()` / `putCN()` 渲染）：

| 页面 | 内容 | 时长 |
|:--|:--|:--|
| 🏠 **标题页** |「反应」+「挑战」+ 闪烁装饰线 +「按下按钮或挥手开始游戏」 | 5 秒自动进入 |
| 🎮 **游戏中** | 方向箭头 / 前推后退 + 倒计时进度条 + 分数/题号 + 连击 + 难度星级 | — |
| 🏆 **结算页** |「游戏结束」/「新纪录」+ 分数/最大连击/已答题数/平均速度/排名 + 上传状态 +「挥手:重试」+「返回:菜单」 | 6 秒后程序停止 |

> `fonts.py` 提供 16px/24px 中文点阵字库，「反应挑战机」四个字从 0 开始逐字设计点阵（16×16 每字 32 字节）。LCD 通过 `printChinese()` 函数渲染汉字，`putCN()` 指定坐标，`putcCN()` 居中显示。

## 六、快速开始

### ESP32 端

将以下文件烧录到 ESP32：

```
display_demo.py
gesture_sensor.py
mqtt_handler.py
fonts.py
```

```python
import display_demo
display_demo.run()
```

程序自动连接 WiFi → 标题页 → 游戏 → 结算 → 停止。

> **WiFi 配置：** 编辑 `mqtt_handler.py` 第 11-12 行，修改 `WIFI_SSID` 和 `WIFI_PASS`。
>
> **MQTT 依赖：** `mqtt_handler.py` 依赖 `umqtt.simple` 模块（MicroPython）。

### Web 排行榜

排行榜已部署于 **GitHub Pages**，`docs/index.html` 为部署入口：

> 🔗 **https://yyyy.github.io/smp**

本地调试可用 VS Code Live Server 直接打开 `docs/index.html`，或 `python -m http.server` 在 `docs/` 目录下启动。

## 八、MQTT 协议

| 主题 | 方向 | 用途 |
|:--|:--|:--|
| `/reaction/score` | ESP32 → Web | 分数上报 |
| `/reaction/control` | Web → ESP32 | 远程指令（已实现但主循环未调用） |
| `/reaction/player` | Web → ESP32 | 玩家名同步 |

**分数 JSON：**
```json
{"device":"ESP32_Reaction","score":147,"mode":"gesture","combo":8,"avg_ms":340,"questions":18}
```

| 字段 | 说明 |
|:--|:--|
| `device` | 设备名（玩家自定义或默认 `ESP32_Reaction`） |
| `score` | 最终得分 |
| `mode` | 固定 `"gesture"` |
| `combo` | 最大连击数 |
| `avg_ms` | 当前固定 340（预留位，未实际测量） |
| `questions` | 答题总数 |

| 前推 | `FORWARD` | 距离题 |
| 后退 | `BACKWARD` | 距离题 |

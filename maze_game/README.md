# 🏃 迷宫游戏 Maze Game

> 2.4寸 LCD 上的随机迷宫闯关游戏，按键操控，5关递进。

---

## 一、项目概述

绿色圆点从左上角出发，穿过蓝色迷宫，到达右下角红色终点即过关。每关迷宫随机生成，每次都是新地图。5关难度递进，按 RESTART 可以重来。

| | |
|:--|:--|
| 🎲 **随机迷宫** | 迭代 DFS 随机回溯算法，每次生成不同迷宫 |
| 📈 **5关递进** | 7×9 → 13×17，越往后路越绕 |
| 🎮 **实体按键** | 上/下/左/右 + RESTART，纯物理操控 |
| 📟 **LCD 渲染** | 墙线绘制 + HUD 信息栏 + 场景过渡动画 |

---

## 二、硬件清单

| 模块 | 型号 | 数量 | 用途 |
|:--|:--|:--|:--|
| 主控 | ESP32 (pyWiFi) | 1 | 主控 |
| 显示屏 | 2.4" TFT LCD (ILI9341) | 1 | 游戏画面 |
| 按键 | 微动开关 | 5 | 方向 + 确认 |

### 接线表

| 按键 | ESP32 引脚 | 接法 |
|:--|:--|:--|
| 上 | GPIO 13 | 一端 GND，一端 GPIO |
| 下 | GPIO 12 | 一端 GND，一端 GPIO |
| 左 | GPIO 14 | 一端 GND，一端 GPIO |
| 右 | GPIO 27 | 一端 GND，一端 GPIO |
| RESTART | GPIO 26 | 一端 GND，一端 GPIO |

> 所有按键使用 ESP32 内部上拉 (`Pin.PULL_UP`)，按下时为低电平，无需外接电阻。

## 四、迷宫生成算法

采用**迭代 DFS + 随机回溯**（非递归，避免 MicroPython 爆栈）：

1. 初始化全墙网格（每格四面都有墙）
2. 从 (0,0) 出发，维护一个栈
3. 随机选一个未访问邻居 → 拆掉中间的墙 → 压栈
4. 无路可走时弹栈回溯
5. 栈空 = 迷宫生成完毕
6. 起点 (0,0) 开北墙作入口，终点开南墙作出口

每个格子的墙壁用**位掩码**存储：

```
bit 0 (1): 北墙
bit 1 (2): 南墙
bit 2 (4): 东墙
bit 3 (8): 西墙
```

附带 BFS 求解器 (`maze.solve()`)，可获取最短路径（调试/演示用）。

## 六、关卡参数

| 关卡 | 迷宫尺寸 | 格子大小 | 大致耗时 |
|:--|:--|:--|:--|
| Lv.1 | 7×9 | ~24px | 15-30 秒 |
| Lv.2 | 9×11 | ~20px | 30-60 秒 |
| Lv.3 | 10×13 | ~18px | 45-90 秒 |
| Lv.4 | 11×14 | ~17px | 60-120 秒 |
| Lv.5 | 13×17 | ~14px | 90-180 秒 |

> 格子大小自适应屏幕，迷宫越大格越小。第 5 关以后保持最大尺寸，可以无限玩。

## 七、核心操作

| 按键 | 标题画面 | 游戏中 | 通关画面 |
|:--|:--|:--|:--|
| 上/下/左/右 | — | 移动玩家 | — |
| RESTART | 开始游戏 | 重新生成当前关 | 进入下一关 |

- 按键带 30ms 消抖
- 边沿触发：按一下走一格，长按不会连续移动
- RESTART 等待按键释放后才响应下一次

---

## 八、使用方法

1. 按接线表焊好 5 个按键
2. 将 `maze_game/` 目录下所有 `.py` 文件复制到 ESP32 根目录
3. 上电或运行 `main.py`
4. 标题画面按 RESTART 开始

## 九、配置修改

改 [config.py](config.py) 即可：

```python
# 改按键引脚
PIN_UP = 13

# 改关卡数量/大小
LEVELS = [(7, 9), (9, 11), (10, 13)]  # 改成3关也行

# 改颜色
WALL_COLOR = (20, 40, 120)  # 迷宫墙壁颜色
```

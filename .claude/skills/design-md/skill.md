## How to Use

When generating UI, reference the DESIGN.md:

```
"Build a login page that looks like Stripe"
"Make this dashboard in the style of Apple"
"Redesign this page with Tesla-inspired aesthetics"
```

I will read the DESIGN.md for that brand and apply its:
- Color palette / gradients
- Typography scale
- Spacing rules
- Border radius / shadows
- Component patterns
- Interaction states
- Tone and copywriting style

## File Structure

Each brand folder contains:
- `DESIGN.md` — Design tokens, component specs, and visual rules
- `README.md` — Summary and usage notes
